# Document Map

Mapa orientativo de secciones para documentos tecnicos largos. No es un router rigido; sirve para priorizar busqueda y validacion.

- Portada y control de versiones: primeras paginas.
- Indice: bloque inicial tras portada.
- Interoperabilidad / ETI: capitulos tecnicos tempranos o medios.
- Impacto ambiental: capitulo especifico o bloque tematico dedicado.
- Descripcion tecnica del proyecto: cuerpo central del documento.
- Presupuesto: tramo final antes de conclusiones o anexos.
- Conclusiones y firmas: cierre del documento.
- Apendices y anexos: ultimas paginas o adjuntos referenciados.

Uso recomendado:

- Buscar portada, indice y firmas antes de lanzar checks estructurales.
- Usar este mapa para acotar `page_start` y `page_end` en evidencia.
- Si el indice contradice el orden real del PDF, priorizar la evidencia observada en paginas.
